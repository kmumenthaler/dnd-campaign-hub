
# Templates & Migration

Purpose

Templates drive the autogenerated note content (PC/NPC/item templates, stat blocks, etc.). The migration system ensures existing notes are updated safely when templates change.

Where to look

- Templates: [src/templates.ts](src/templates.ts)
- Migration types & versions: [src/migration/types.ts](src/migration/types.ts)
- Migration registry: [src/migration/registry.ts](src/migration/registry.ts)
- Migration runner: [src/migration/runner.ts](src/migration/runner.ts)
- Frontmatter utilities: [src/migration/frontmatter.ts](src/migration/frontmatter.ts)
- Migration modal: [src/migration/MigrationModal.ts](src/migration/MigrationModal.ts)

Rules & checklist

- Increment the `template_version` inside the template string whenever you change displayed or inserted content.
- Update `TEMPLATE_VERSIONS` in `src/migration/types.ts` to match the new version.
- Add a `MigrationStep` in `src/migration/registry.ts` for non-trivial changes that add new blocks or UI (never remove user data).
- Make migrations idempotent: check whether the new content already exists before inserting.
- Migration steps are pure string transforms — they receive full file content and return modified content (or `null` to skip).

Minimal migration pattern

1. Add a `MigrationStep` object inside `getAllMigrations()` in `src/migration/registry.ts`.
2. The step must specify `entityType`, `targetVersion`, `description`, and an `apply()` function.
3. In `apply()`, parse frontmatter with `parseFrontmatter()`, check if the change already exists, then apply content transforms using helpers from `frontmatter.ts`.
4. Always call `setFrontmatterField(content, "template_version", targetVersion)` as the last step.

Example

```typescript
// inside getAllMigrations() in registry.ts
{
  entityType: "player",
  targetVersion: "1.4.0",
  description: "Add new-field to PC notes",
  apply(content: string, ctx: MigrationContext): string | null {
    const fm = parseFrontmatter(content);
    if (!fm) return null;

    if (fm.fields["new_field"] !== undefined) {
      return setFrontmatterField(content, "template_version", "1.4.0");
    }

    let out = content;
    out = addFrontmatterFieldAfter(out, "existing_field", "new_field", "default_value");
    out = setFrontmatterField(out, "template_version", "1.4.0");
    return out;
  },
},
```

Testing & deployment

- Build and run the plugin locally and test migration on a copy of a campaign.
- `MigrationRunner` creates automatic backups in `.dnd-hub-backups/` before applying changes.
- Use the Migration Modal (Ctrl+P → "Run Template Migration") to preview and apply migrations.
